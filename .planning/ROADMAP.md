# Roadmap: Focus Reps

## Overview

Transform the existing Peak Mind codebase into Focus Reps by rebranding, replacing all practice content with original scripts, and adding deliberate‑practice feedback loops and weekly review insights. The roadmap prioritizes IP‑safe content, then metrics and review loops, then notifications and release readiness for iOS and Android.

## Phases

- [x] **Phase 1: Rebrand + Content Foundation** - Rename practices, rewrite scripts, and reframe onboarding
- [x] **Phase 2: Practice Progression & Schedule Control** - Solidify schedule logic, flexibility, and longer training options
- [x] **Phase 3: Feedback & Metrics Capture** - Daily survey, lapse counts, and uninterrupted interval tracking
- [ ] **Phase 4: Weekly Review & Recommendations** - Aggregate metrics + journaling highlights into weekly insights
- [ ] **Phase 5: Notifications & Release Readiness** - Weekly summary reminder + cross‑platform polish

## Phase Details

### Phase 1: Rebrand + Content Foundation
**Goal**: Establish Focus Reps identity with original practice content and onboarding narrative.
**Depends on**: Nothing (first phase)
**Requirements**: BRAND-01, CONTENT-01, CONTENT-02, ONBOARD-01
**Success Criteria**:
  1. User sees Focus Reps branding and copy on all primary screens.
  2. All practices display new names and original scripts (audio + text).
  3. Onboarding explains deliberate‑practice focus and mentor framing.
**Plans**: 3 plans

Plans:
- [x] 01-01: Rebrand UI + copy updates across screens
- [x] 01-02: Rename practices + replace scripts
- [x] 01-03: Update onboarding narrative

### Phase 2: Practice Progression & Schedule Control
**Goal**: Support structured 6‑week program with flexibility, practice selection, and extended training option.
**Depends on**: Phase 1
**Requirements**: SCHED-01, SCHED-02, SCHED-03, SCHED-04
**Success Criteria**:
  1. User can follow the 6‑week schedule by default.
  2. User can continue practice after week 4 with flexible pacing.
  3. User can select a practice outside the schedule.
  4. User can opt into a longer training option (e.g., 8‑week path).
**Plans**: TBD (2–3 plans)

Plans:
- [x] 02-01: Schedule engine updates (6‑week + flexible pacing)
- [x] 02-02: Practice selection UX + extended training option

### Phase 3: Feedback & Metrics Capture
**Goal**: Implement deliberate‑practice feedback loop with surveys, lapses, and uninterrupted interval tracking.
**Depends on**: Phase 2
**Requirements**: FEED-01, FEED-02, METRIC-01, METRIC-02, PRIV-01
**Success Criteria**:
  1. User completes a daily survey after practice (focus, mood, stress, energy).
  2. User records post‑session lapse count.
  3. App captures longest uninterrupted focus interval per session.
  4. Session minutes + reps summary are stored locally.
  5. All feedback data is stored locally (no account required).
**Plans**: TBD (2–3 plans)

Plans:
- [x] 03-01: Session model + storage updates for metrics
- [x] 03-02: Daily survey + lapse capture UI
- [x] 03-03: Uninterrupted interval tracking

### Phase 4: Weekly Review & Recommendations
**Goal**: Deliver a weekly review that summarizes focus gains and recommends next practice.
**Depends on**: Phase 3
**Requirements**: REVIEW-01, REVIEW-02, REVIEW-03, REVIEW-04, REVIEW-05
**Success Criteria**:
  1. Weekly review shows total focus minutes.
  2. Weekly review shows longest uninterrupted interval.
  3. Weekly review shows lapses per session.
  4. Weekly review highlights journal entries from the week.
  5. Weekly review recommends the next practice.
