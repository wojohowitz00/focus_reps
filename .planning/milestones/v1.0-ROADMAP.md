# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-02-04
**Phases:** 1-5
**Total Plans:** 12

## Overview

Transform the existing Peak Mind codebase into Focus Reps by rebranding, replacing all practice content with original scripts, and adding deliberate‑practice feedback loops and weekly review insights. The roadmap prioritized IP‑safe content, then metrics and review loops, then notifications and release readiness for iOS and Android.

## Phases

### Phase 1: Rebrand + Content Foundation

**Goal**: Establish Focus Reps identity with original practice content and onboarding narrative.
**Depends on**: Nothing (first phase)
**Plans**: 3 plans

Plans:

- [x] 01-01: Rebrand UI + copy updates across screens
- [x] 01-02: Rename practices + replace scripts
- [x] 01-03: Update onboarding narrative

**Details:**
- User sees Focus Reps branding and copy on all primary screens.
- All practices display new names and original scripts (audio + text).
- Onboarding explains deliberate‑practice focus and mentor framing.

### Phase 2: Practice Progression & Schedule Control

**Goal**: Support structured 6‑week program with flexibility, practice selection, and extended training option.
**Depends on**: Phase 1
**Plans**: 2 plans

Plans:

- [x] 02-01: Schedule engine updates (6‑week + flexible pacing)
- [x] 02-02: Practice selection UX + extended training option

**Details:**
- User can follow the 6‑week schedule by default.
- User can continue practice after week 4 with flexible pacing.
- User can select a practice outside the schedule.
- User can opt into a longer training option (8‑week path).

### Phase 3: Feedback & Metrics Capture

**Goal**: Implement deliberate‑practice feedback loop with surveys, lapses, and uninterrupted interval tracking.
**Depends on**: Phase 2
**Plans**: 3 plans

Plans:

- [x] 03-01: Session model + storage updates for metrics
- [x] 03-02: Daily survey + lapse capture UI
- [x] 03-03: Uninterrupted interval tracking

**Details:**
- Daily survey after practice (focus, mood, stress, energy).
- Post‑session lapse count recorded.
- Longest uninterrupted focus interval tracked per session.
- Session minutes + reps summary stored locally.
- All feedback data stored locally (no account required).

### Phase 4: Weekly Review & Recommendations

**Goal**: Deliver a weekly review that summarizes focus gains and recommends next practice.
**Depends on**: Phase 3
**Plans**: 2 plans

Plans:

- [x] 04-01: Weekly aggregation logic
- [x] 04-02: Weekly review UI + recommendations

**Details:**
- Weekly review shows total focus minutes.
- Weekly review shows longest uninterrupted interval.
- Weekly review shows lapses per session.
- Weekly review highlights journal entries from the week.
- Weekly review recommends the next practice.

### Phase 5: Notifications & Release Readiness

**Goal**: Enable weekly summary reminder and validate cross‑platform readiness.
**Depends on**: Phase 4
**Plans**: 2 plans

Plans:

- [x] 05-01: Weekly/daily reminder scheduling
- [x] 05-02: Cross‑platform QA + polish

**Details:**
- Weekly summary reminder is available (opt‑in).
- Optional daily reminder can be enabled or disabled.
- Core flows work on iOS and Android.

## Milestone Summary

**Decimal Phases:**

- None.

**Key Decisions:**

- Use Focus Reps as the visible product name across primary screens.
- Rename practices to Anchor Breath, Body Sweep, Thought Traffic, Kindness Circuit with original scripts.
- Normalize legacy practice IDs on read to preserve existing session history.
- Add program modes (standard 6‑week, extended 8‑week, open training) with program-aware scheduling.
- Weekly reminders default off (Sunday 19:00) with opt‑in scheduling.
- Weekly review aligns to the current program week via `getCurrentWeek`.

**Issues Resolved:**

- Removed Peak Mind branding and IP‑linked practice content.
- Implemented deliberate‑practice feedback loop (survey + lapses + longest interval).
- Delivered weekly review with recommendations and journal highlights.

**Issues Deferred:**

- Device QA for iOS/Android notification flows.
- Confirm decision on daily reminders vs. weekly-only.

**Technical Debt Incurred:**

- Weekly reminder rescheduling can duplicate notifications if day/time changes.
- PROJECT.md decision drift and Active requirement list not updated.
